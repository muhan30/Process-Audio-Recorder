# SPEC: 低通滤波降噪 + 文件大小显示修正

> 版本: 2.0 | 日期: 2026-07-20 | 状态: 待审查
> v1→v2 变更: 噪声门方案 → 低通滤波方案（用户反馈：用频率过滤高频噪声更合理）

---

## 一、背景

用户实战录音后发现两个问题：

1. **文件大小"虚标"**：M4A 格式录音时，显示的文件大小约 10 秒 2MB，但磁盘上实际文件只有 ~200KB。原因是显示的是压缩前的 PCM 数据量，不是压缩后的实际文件大小。
2. **电流高频噪声**：麦克风增益 ×4.0 把麦克风自身底噪（电路白噪声）放大了 4 倍，录音中持续有"嘶嘶"高频声。

---

## 二、需求

### 需求 1：M4A 文件大小显示真实值

**目标**：GUI 上显示的文件大小 = 磁盘上实际文件大小。

**验收标准**：
- M4A 格式录音时，文件大小显示应与 Windows 资源管理器中看到的文件大小基本一致（误差 < 10%，因为编码器有缓冲）
- WAV 格式行为不变（PCM 数据量 = 文件大小，无需修改）
- 显示格式不变：< 1MB 显示 KB，≥ 1MB 显示 MB（一位小数）

### 需求 2：低通滤波器（Low-Pass Filter）——降高频噪声

**目标**：滤掉麦克风信号中的高频电流噪声，同时不损伤语音。

**为什么用低通滤波而不是噪声门**：

| | 噪声门 | 低通滤波 |
|------|------|------|
| 不说话时 | ✅ 切静音，无噪声 | ✅ 高频噪声被滤掉 |
| 说话时 | ❌ 噪声和语音一起通过 | ✅ 噪声持续被滤掉，语音保留 |
| 副作用 | 噪声忽有忽无（抽吸感） | 无（平滑滤波） |

**原理**：

人声的频率分布：
- 基频：85~500 Hz
- 泛音（音色）：~4 kHz
- 齿音（s/sh/f）：~8 kHz
- **8 kHz 以上几乎没有语音能量**

电流噪声的频率分布：
- 电路白噪声覆盖全频段
- 最刺耳的"嘶嘶"声集中在 **8~20 kHz**

→ 用一个截止频率 10 kHz 的低通滤波器，把 10 kHz 以上的高频一刀切掉，嘶声没了，语音毫发无损。

```
麦克风 PCM 采样
      │
      ▼
  低通滤波器 (fc=10kHz, 二阶巴特沃斯)
      │
      ├── ≤ 10kHz: 正常通过（含全部语音 + 低频噪声，但低频噪声人耳不敏感）
      │
      └── > 10kHz: 大幅衰减（嘶声在此频段，被滤掉）
      │
      ▼
  送入混音器
```

**巴特沃斯滤波器特点**：
- 通带内（< 10kHz）完全平坦，不影响音色
- 阻带每倍频程衰减 12dB，10kHz 以上噪声迅速消失
- 二阶（12dB/oct）是音频领域最常用的降噪滤波器阶数

**验收标准**：
- 录音中高频"嘶嘶"声明显减少或消失
- 说话声音正常，齿音（s/sh）不被削弱
- 滤波器只作用于麦克风路径，不影响系统声音（微信对方语音）
- 截止频率默认 10.0 kHz
- 可在 INI 中配置（`[Settings]` → `LowpassCutoff`），值为 0 表示关闭滤波器
- 截止频率在设置对话框中可调（0 = 关闭，4.0 ~ 20.0 kHz）

## 四、数据流

### 文件大小（M4A）

```
之前: M4aSink::BytesWritten() → m_cbPcmWritten（PCM 字节，偏大 10 倍）
之后: M4aSink::BytesWritten() → _wstat64(文件路径).st_size（磁盘实际大小）
```

### 低通滤波

```
之前: MicCapture → PushMicData → MixWithLoopback（无条件混入）

之后: MicCapture → PushMicData → MixWithLoopback
                                         │
                                   取麦克风数据块
                                         │
                                   低通滤波（双声道各自处理）
                                         │
                                   滤波后混入环回流
```

## 五、滤波器实现细节

### 二阶巴特沃斯低通（Biquad）

每个声道维护 4 个状态变量（x[n-1], x[n-2], y[n-1], y[n-2]）。

```
差分方程:
y[n] = b0*x[n] + b1*x[n-1] + b2*x[n-2] - a1*y[n-1] - a2*y[n-2]
```

**系数表（44.1kHz 采样率下的预计算值）**：

| 截止频率 | 说明 | 效果 |
|---------|------|------|
| 12 kHz | 极轻 | 只滤超声波，保留一切可听细节 |
| **10 kHz（默认）** | 标准 | 滤嘶声，保留全部语音 |
| 8 kHz | 偏强 | 轻微削弱齿音，噪声更少 |
| 6 kHz | 强 | 明显降噪，语音略闷 |
| 0 | 关闭 | 不过滤 |

### 为什么不用 FFT

巴特沃斯 IIR 滤波器每个采样只需 5 次乘加运算，对双声道 44.1kHz 的 CPU 占用可忽略。FFT 需要分帧→加窗→FFT→频域衰减→IFFT→叠接，计算量大、有延迟、代码复杂——对实时录音降噪是大材小用。

## 六、测试计划

1. **M4A 文件大小**：录 30 秒，每 10 秒对比显示值和资源管理器实际值
2. **WAV 文件大小**：同上，确认未受影响
3. **滤波效果**：录一段"不说话"的静音段，对比滤波前后频谱/听感
4. **语音保留**：录一段说话（含 s/sh 齿音），确认齿音不丢
5. **截止频率=0**：确认关闭后行为与之前一致（旁路滤波器）
6. **编译**：CLI + GUI 两个 target 均通过
