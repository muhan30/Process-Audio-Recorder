# 阶段二 GUI 包装·规格书

- 日期：2026-07-18
- 依赖：阶段一引擎全部完工（`docs/superpowers/specs/2026-07-17-process-recorder-app-design.md` 第 6 节阶段二部分）
- 上游设计：头脑风暴三轮确认的布局、托盘行为、设置项、意外处理方案

## 1. 目标

把阶段一做好的命令行引擎包一层 Windows 原生图形界面，变成可双击运行、无需敲命令的自用录音工具。

## 2. 不做什么

阶段一的设计文档里列了"第二版才做"的东西，本阶段一概不碰：一键预设、分轨保存、录音管理/回放列表、开机自启、全局快捷键、安装包、自动更新。

## 3. 界面清单

### 3.1 主窗口（之前画过 mockup，你已确认）

**待机状态：**
- 标题栏：🎙️ 进程录音机
- "正在发声的软件"列表，每行显示：图标 / 进程名 / exe 名 / 🔊 正在发声 或 安静
- 列表顶部有 **刷新** 按钮
- 列表下面一行提示：💡 没看到想录的软件？让它先出个声，再点刷新
- ☑ **同时录我的麦克风**（默认勾上）
- 大红色 **● 开始录音** 按钮
- 底部：保存到 `recordings\`，旁边有 **打开文件夹** 链接

**录音中状态：**
- 标题栏变为 🎙️ 进程录音机 - 录音中
- 计时区显示：正在录制：微信 / **00:12:34** / 文件大小
- 🔊 系统声音 电平条（实时跳动）
- 🎤 麦克风 电平条（实时跳动）
- 大灰色 **⏹ 停止并保存** 按钮

### 3.2 设置窗口（齿轮 ⚙ 按钮打开）

三样东西：
1. **保存文件夹**：输入框 + 浏览按钮，默认填 `recordings/`（可自由改）
2. **输出格式**：两个单选按钮，"M4A 压缩（推荐）"和"WAV 无损"
3. **麦克风设备**：下拉框列出所有麦克风
4. **增益**：两个滑块（做完 1-3 后追加）

### 3.3 托盘图标

- 点主窗口 ✕ → 缩到右下角托盘，**录音不中断**
- 托盘图标：录音中叠加红点
- 双击图标 → 恢复主窗口
- 右键菜单：打开主窗口 / 停止录音 / 退出
- 退出程序时如果正在录音弹确认框

### 3.4 增益调节（你在阶段一实测确认的需求）

从设置窗口或主窗口底部的快捷区：
- **系统声音增益滑块**：范围 0.1-8.0，默认 **2.0**
- **麦克风增益滑块**：范围 0.1-8.0，默认 **4.0**
- 拖动时如果有录音在进行，实时生效（听感即时反馈）

## 4. 操作流程

**打开软件 → 选目标 → 开录 → 缩托盘 → 回来停止：**

1. 双击 exe → 主窗口出现，自动刷新"正在发声的软件"列表
2. 找到微信（它会标 🔊），点它选中；确保"录麦克风"勾着
3. 点 **开始录音** → 按钮变灰、计时开始、双电平条开始跳
4. 把窗口 ✕ 掉 → 缩到托盘，图标出红点，录音继续
5. 通话结束 → 双击托盘图标恢复窗口 → 点 **停止并保存**
6. 文件落在 `recordings\` 里，命名格式 `2026-07-18 微信 22-30.m4a`
7. 底部链接点"打开文件夹"直接跳过去

## 5. 意外处理（引擎层已有的 + GUI 层的补充）

| 情况 | GUI 怎么反应 |
|------|-------------|
| 目标进程退出 | 弹气泡提示"微信已退出，录音已自动保存"，正常收尾 |
| 列表里没有微信 | 提示"没看到想录的软件？让它先出个声，再点刷新" |
| 麦克风初始化失败 | 弹框问"麦克风打不开，只录对方声音继续？还是取消？" |
| 磁盘满 | 弹框警告，已录部分尽量保住 |
| 录音中退出程序 | 弹确认框"正在录音，确定退出？" |
| 录音中系统要睡眠 | 录音期间自动阻止睡眠（引擎层已做） |

## 6. 开发方式

所有引擎功能（`--list`、麦克风、混音、M4A、电平）已在阶段一的命令行里调试稳定。GUI 层只需要：
1. 创建窗口、画控件
2. 调引擎接口取数据（发声列表、电平数值）
3. 把用户的点击翻译成引擎指令（开始/停止/增益值）

引擎一行不改——它已经在阶段一验证通过了。

## 7. 文件输出

- 文件名：`<日期> <目标名> <时间>.m4a`（例：`2026-07-18 微信 22-30.m4a`）
- 全局模式（mode 0）用"系统混音"作目标名
- 扩展名按设置里的格式选择（默认 .m4a）
- 上次使用的保存目录记下来，下次打开沿用

## 8. 验收标准

每一条你亲手验：

1. **列表正确**：酷狗放歌 → 打开软件 → 列表里酷狗标着 🔊 排第一个
2. **录音操作闭环**：选酷狗 → 开始 → 电平条跳 → 停止 → 文件生成且能听
3. **混音 + 增益生效**：勾麦克风 → 默认增益 2/4 → 说话+放歌 → 一个文件两人声音
4. **托盘行为**：录音中点 ✕ → 缩到托盘 → 红点 → 双击恢复 → 停止正常保存
5. **退出保护**：录音中点托盘退出 → 弹出确认框 → 确认后才退出并保存
6. **设置持久化**：改了保存路径或格式 → 关掉重开 → 还记着上次的选择
7. **防空录**：选 quiet 进程 → 电平条不动
8. **单 exe**：产物就是一个 .exe，双击就跑，不依赖安装

## 9. 阶段二说明

本次 GUI 实现范围严格按本 SPEC —— 额外功能（如实时增益滑块、麦克风静音快捷切换、录音时长预警、自动化定时录制等）除非用户明确要求均不纳入了。其目的仅仅是让该录音引擎有一个可双击运行的界面，而非构建完善产品。

SPEC 中标记为"后续"的项目，将视实际使用中发现的痛点再做是否需要、值得的阶段 2.x 规划。
