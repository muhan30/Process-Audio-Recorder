# SPEC: 噪声门 + 按软件名分文件夹

> 版本: 1.0 | 日期: 2026-07-20 | 状态: 待审查

---

## 一、背景

用户测试低通滤波（默认 10kHz）后反馈：音频质量的改善明显，但背景仍存在中低频电流噪声。此外，1kHz 截止频率虽能滤掉更多噪声，但会损伤语音清晰度。

**结论**：单靠低通滤波无法完全消除噪声——电流噪声覆盖全频段，低通只能砍高频部分。需要噪声门在"不说话时"彻底静音。

同时，随着录音次数增多，recordings/ 下文件混杂，需要按软件名自动分文件夹。

---

## 二、需求

### 需求 1：噪声门（Noise Gate）

**目标**：不说话时麦克风完全静音，消除所有频段的底噪。

**原理**：

```
麦克风数据块
      │
      ▼
  RMS 电平计算
      │
      ├── RMS < 阈值 dBFS ──→ 整块置零（不说话，静音）
      │
      └── RMS ≥ 阈值 dBFS ──→ 低通滤波 → 混入（说话，频噪被滤）
```

**为什么放在低通之前**：噪声门判断"有没有在说话"只看信号强度不看频率。先判断是否静音（全频段切掉），再对通过的信号做低通（砍高频嘶声）。两者互补：
- 噪声门：解决"不说话时的全频段噪声"
- 低通滤波：解决"说话时的高频嘶声"

**验收标准**：
- 不说话时录音完全安静（所有频段噪声被门挡住）
- 小声说话不被误切（阈值设在噪声和语音之间）
- 说话到安静过渡平滑，无"啪"的切断声
- 阈值可调，范围 -60 ~ -20 dBFS
- 默认 -45 dBFS（适合大多数桌面麦克风）
- 设为 0 关闭噪声门
- INI 持久化：`[Settings] NoiseGate`，存储为整数 = |dBFS| × 10（如 -45 → 450）

**dBFS 参考**：
| 阈值 | 适用场景 |
|------|---------|
| -60 | 非常安静的录音棚 |
| -50 | 安静的桌面环境 |
| **-45（默认）** | 普通桌面麦克风，有轻微底噪 |
| -35 | 嘈杂环境（风扇/空调声大） |
| -20 | 几乎不降噪，只挡极低电平 |
| 0 | 关闭 |

### 需求 2：录音按软件名分文件夹

**目标**：`recordings/` 下每个软件一个子文件夹。

**验收标准**：
```
之前: recordings/2026-07-20 Weixin 14-30-00.m4a
      recordings/2026-07-20 Chrome 14-32-15.m4a
      recordings/2026-07-20 Weixin 15-10-22.m4a

之后: recordings/Weixin/2026-07-20 Weixin 14-30-00.m4a
      recordings/Chrome/2026-07-20 Chrome 14-32-15.m4a
      recordings/Weixin/2026-07-20 Weixin 15-10-22.m4a
```
- 子文件夹在录音开始时自动创建（`CreateDirectory`）
- 文件名格式不变
- 如果进程名含非法字符（如 `CON`、`LPT1`），自动替换为下划线

## 四、实现概要

### 噪声门

- AudioMixer 新增 `m_noiseGateThreshold`（atomic float），默认 -45.0f
- 新增 `SetNoiseGateThreshold(float dBFS)`，0 = 关闭
- MixWithLoopback 中，取 mic 数据后：
  1. 计算 RMS：`sqrt(mean(x²))` 对所有样本
  2. 转为 dBFS：`20 * log10(RMS / 32768)`
  3. 如果 < 阈值 → `memset(micChunk, 0)`
  4. （如果通过）→ 低通滤波 → 混音
- RMS 计算只在 `micChunk` 非全零时执行

### 分文件夹

- OnStart 中构建路径时，`recDir + L"\\" + procName` 作为子文件夹
- `CreateDirectory` 创建子文件夹
- 文件名保持 `日期 进程名 时间.格式`

### GUI 设置

- 设置对话框新增一行"噪声门 (dBFS)，0=关闭"，默认显示 "-45.0"
- 窗口高度 260 → 300
- 输入校验：范围 -60.0 到 -20.0，或 0
